hexo部署到云服务器

发表于2021-02-18|更新于2021-06-28|记录

准备好：

  • 云服务器
  • 域名

在本地hexo博客生成

  • 本地系统是 Windows10

前置工具安装：

  • Node.js

    选择12版本的，新的版本hexo会报错

  • Git

    点击Download

如果能看到版本号，就算安装好了(在命令行使用之前，须要加入相应的环境变量)

hexo 安装

npm install -g hexo-cli

能看到版本号就算安装成功了

新建一个文件夹作为博客目录，例如：hexo

进入hexo目录后依次执行：

hexo init
npm install

然后就可以执行hexo命令了：

hexo cl    #清理缓存
hexo g    #渲染博客文件
hexo s    #预览本地渲染好的博客
hexo d    #部署至远端 (需要配置部署选项)

按住Ctrl键单机地址，会自动打开浏览器，在本地预览自己的博客。

hexo部署配置

在 hexo 配置文件最后

deploy:
    type: git
    repo: git@IP或域名:远程仓库路径
# 如果修改了ssh端口的话，需要这么写：
#    repo: ssh://git@IP或域名:端口号/远程仓库路径
    branch: master

如果需要部署到多个远端(比如：服务器和GitHub)，需要这么写：

deploy:
    type: git
    repo: 
        github: git@github:GitHub用户名/远程仓库路径
        server: git@IP或域名:远程仓库路径
    branch: master

服务器上配置

  • 系统是 Ubuntu

创建一个用户，以后博客推送就通过这个用户

useradd git -m    # git是示例，其他名字也行

创建 Git 仓库：

mkdir /var/repo
cd /var/repo
git init --bare hexo.git

创建钩子：

cd /var/repo/hexo.git/hooks
vim post-receive
---------把下面一句内容复制进去-----------
git --work-tree=/var/www/hexo --git-dir=/var/repo/hexo.git checkout -f
------------------------------------------
#第一段路径是网站路径，第二段是Git仓库路径

nginx配置

配置证书就不说了，阿里云文档里面有教程

改一下 nginx 用户

vim /etc/nginx/nginx.conf
# 在前几行找到 user ，把user后面的用户改成 root

然后执行：

chown -R git:git /var/repo
chmod -R 700 /var/repo
chown -R git:git /var/www
chmod -R 700 /var/www

把 Git 仓库用户改为 git 以确保推送时没有权限问题。因为推送时 SSH 使用的用户是 git。

最终博客文件是放到 www 目录了，所以 www 目录也需要有 git的权限。否则推送时会报无权限。

在本地使用 Git 生成密钥对

密钥对用来免密码登录服务器、免密码向服务器推送你的博客文件，很方便

本地执行

在本地命令行输入：

ssh-keygen

一路回车~~~后，就会看到这样的消息，并且在用户目录内生成了一个.ssh文件，里面就是密钥对：

打开后缀.pub文件，复制里面全部内容

id_rsa 文件要保存好

服务器上执行

在 git 用户根目录创建：

mkdir .ssh
vim authorized_keys
# 然后把刚刚在本地复制的公钥的内容粘贴进去，保存

改权限：

chmod 700 .ssh
chmod 400 .ssh/authorized_keys # 防止被修改

如果想给其他用户也配置免密码登录，就修改相应用户目录内的.ssh文件就行。

登录测试

ssh git@IP地址 -p 端口号
# 如果没有修改ssh端口号，就去掉-p

输入yes后，如果没有需要输入密码的情况，就证明可以免密登录了。

登录脚本配置

如果喜欢用终端远程登录的话，配置个登录脚本会更方便

在本地.ssh文件夹内创建congfig文件

写入下面的配置：

Host XXX    #执行标本别名
HostName 服务器IP地址
Port 端口号
User git    #需要登录的用户
IdentityFile C:\Users\june\.ssh\id_rsa    #私钥文件 id_rsa 存放路径

配置好后在命令行上直接输入下面的命令登录：

ssh XXX    #上面自己起的别名

配置完成
